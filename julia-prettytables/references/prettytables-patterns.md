# PrettyTables Patterns

## Quick Backends

```julia
using PrettyTables

pretty_table(data; backend=:text)
md = pretty_table(String, data; backend=:markdown)
html = pretty_table(String, data; backend=:html, stand_alone=true)
```

## Common Report Sections

```julia
pretty_table(
    data;
    title="Model Report",
    subtitle="Validation Set",
    column_labels=["metric", "value"],
    row_labels=["row1", "row2"],
    show_row_number_column=true,
    summary_rows=[sum],
    summary_row_labels=["Total"],
    source_notes="Generated by pipeline"
)
```

Advanced section options:
- `row_group_labels`
- `merge_column_label_cells`
- `footnotes`

## Formatting

Use formatter functions with signature `(v, i, j) -> new_value`:

```julia
fmt = (v, i, j) -> j == 2 ? round(v, digits=3) : v
pretty_table(data; formatters=[fmt])
```

Predefined formatters are available in PrettyTables for common numeric formatting workflows.

## Highlighters

Backend-specific highlighters:
- `TextHighlighter`
- `MarkdownHighlighter`
- `HtmlHighlighter`

Each highlighter uses a predicate on `(data, i, j)` and a style/decoration result.

## Alignment and Cropping

Useful cross-backend controls:
- `alignment` and section-specific alignment keywords
- `maximum_number_of_rows`
- `maximum_number_of_columns`
- `vertical_crop_mode` (`:bottom` or `:middle`)
- `show_omitted_cell_summary`

## Backend Notes

Text backend:
- `table_format::TextTableFormat`
- `style::TextTableStyle`
- supports rich border and terminal styling controls

Markdown backend:
- `allow_markdown_in_cells`
- `line_breaks` (`\n` to `<br>`)
- `table_format::MarkdownTableFormat`

HTML backend:
- `allow_html_in_cells`
- `stand_alone`, `minify`
- `wrap_table_in_div`, `table_class`, `table_div_class`
- `table_format::HtmlTableFormat`, `style::HtmlTableStyle`
